Add all 5 test scenarios to `tests/test_comparison.py` below the helpers. Each test is marked `@pytest.mark.integration`.

**Scenario 1 — Simple navigation (CMP-01):**
`test_scenario1_simple_navigation`
- Proxy path: create session, navigate to https://example.com, snapshot, get_content -> assert "Example Domain" in content.
- Direct path: use direct_client fixture, call_tool browser_navigate, call_tool browser_snapshot, extract `result["content"][0]["text"]` -> assert "Example Domain".
- Both paths see the same landmark content.

**Scenario 2 — Diff behavior (CMP-02):**
`test_scenario2_diff_behavior`
- Proxy path: create session, navigate to example.com, snapshot, get_content (first read: non-empty), get_content again same ref_id (second read: empty string — diff suppression).
- Direct path: navigate, two sequential browser_snapshot calls -> both return non-empty content.
- Assert proxy second read is empty, direct second read is non-empty. This is the key behavioral difference.

**Scenario 3 — Multi-page navigation (CMP-03):**
`test_scenario3_multi_page_navigation`
- Proxy path: create session, navigate to example.com -> snapshot -> get_content (assert "Example Domain"), then navigate to httpbin.org -> snapshot -> get_content (assert "httpbin" case-insensitive). Verify the two ref_ids are different.
- Direct path: navigate to example.com -> snapshot (assert "Example Domain"), navigate to httpbin.org -> snapshot (assert "httpbin" case-insensitive).
- Both paths see correct content for each page.

**Scenario 4 — Content search (CMP-04):**
`test_scenario4_content_search`
- Proxy path: create session, navigate to example.com, snapshot, get_content with `search_for="Example Domain"` and `reset_cursor=true` (filtered result), then get_content with `reset_cursor=true` only (full result). Assert filtered length < full length. Both contain "Example Domain".
- Direct path: snapshot returns full content. Filter manually with Python `"Example Domain" in content`. Assert full content returned (no built-in filtering).
- Demonstrates proxy's search_for is a value-add.

**Scenario 5 — Error handling (CMP-05):**
`test_scenario5_error_handling`
- Proxy path: create session, navigate to `https://this-domain-does-not-exist-at-all-12345.invalid`. Assert either response `status == "error"` OR the proxy returns an error HTTP status. The test must not crash.
- Direct path: call_tool browser_navigate with same invalid URL. Expect either a RuntimeError (MCP error) or a result with `isError: true`. Wrap in try/except to handle both cases gracefully.
- Both paths fail gracefully.

Important implementation notes:
- Use `async with httpx.AsyncClient(base_url=BASE_URL, timeout=TIMEOUT) as client:` for each test's proxy path.
- Use the `direct_client` fixture for direct path.
- Assert on stable landmark content only (never on ref attributes like `[ref=e6]`).
- Each test is independent — no ordering dependencies.
