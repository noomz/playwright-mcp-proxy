**YAML Frontmatter:**
```yaml
name: playwright-proxy
description: >
  Browser automation via playwright-mcp-proxy. Use this skill when navigating websites,
  capturing page content, clicking elements, typing text, or checking console logs
  through the Playwright proxy. The proxy persists all interactions in SQLite and returns
  only metadata + ref_id — always call get_content(ref_id) to retrieve actual page content.
allowed-tools:
  - playwright-proxy
  - read_file
  - write_file
  - glob
  - search_file_content
  - replace
```

**Body sections (in order):**

1. **Prerequisites** (3-4 lines): Server must be running. Start with `playwright-proxy-server` or `uv run playwright-proxy-server`. Verify with `playwright-proxy-ctl health`.

2. **Core Concept: Response Policy** (5-7 lines): All browser action tools return metadata + ref_id only, never full page content. Always call `get_content(ref_id)` after any action that produces a snapshot. This is the standard workflow, not an optional pattern.

3. **Session Lifecycle** (code block showing the flow):
   ```
   create_new_session
       -> browser_navigate(url) -> ref_id
       -> get_content(ref_id) -> page snapshot
       -> browser_click/type(...) -> ref_id
       -> get_content(ref_id) -> changes (or empty if unchanged)
       -> browser_close
   ```

4. **Tool Reference** (table format, all 9 tools):
   Document each tool from the TOOLS list in mcp_server.py with:
   - Tool name
   - Parameters (required marked, optional noted)
   - What it returns

   Tools to document:
   - `create_new_session` — no params, returns session_id string
   - `browser_navigate` — url (required), returns metadata + ref_id
   - `browser_snapshot` — no params, returns metadata + ref_id
   - `browser_click` — element (required), ref (required), returns metadata + ref_id
   - `browser_type` — element (required), ref (required), text (required), submit (optional bool), returns metadata + ref_id
   - `browser_console_messages` — onlyErrors (optional bool), returns metadata + ref_id
   - `browser_close` — no params, returns metadata + ref_id
   - `get_content` — ref_id (required), search_for (optional), reset_cursor (optional bool), before_lines (optional int), after_lines (optional int), returns page snapshot text
   - `get_console_content` — ref_id (required), level (optional: debug/info/warn/error), returns console log text

5. **Diff Behavior** (5-7 lines): `get_content` returns empty string if page unchanged since last call (hash-based diff). First read returns full content. Subsequent reads return empty if no change, full content if changed. Use `reset_cursor=true` to force full content. Console logs do NOT use diff.

6. **Content Search** (3-4 lines): `get_content` supports `search_for` parameter for grep-like filtering. Use `before_lines` and `after_lines` for context (like grep -B/-A).

7. **Management CLI** (code block):
   ```bash
   playwright-proxy-ctl health                          # server status
   playwright-proxy-ctl sessions list                   # all sessions
   playwright-proxy-ctl sessions list --state active    # filter by state
   playwright-proxy-ctl sessions clear                  # clear closed (with prompt)
   playwright-proxy-ctl sessions clear --state error --yes  # skip prompt
   playwright-proxy-ctl db vacuum                       # compact DB (server must be stopped)
   ```

8. **Test Generation Workflow** (preserve backward compat, 5-7 lines): Brief section noting that for Playwright test spec generation, configure `.claude-playwright.yml` with `spec_dir` and `playwright_dir`. Walk through each step with browser tools before writing test code. Keep session open for user confirmation.
