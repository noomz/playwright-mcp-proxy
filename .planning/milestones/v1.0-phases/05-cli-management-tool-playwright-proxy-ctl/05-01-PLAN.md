    - health: server down -> prints "not running" to stderr, exit 1
    - health: timeout -> prints "timed out" to stderr, exit 1
    - sessions list: server returns sessions -> prints each session (truncated ID, state, URL), total count, exit 0
    - sessions list: no sessions -> prints "No sessions found.", exit 0
    - sessions list --state active: passes state param in query string
    - sessions list: server down -> prints error, exit 1
    - sessions clear --state closed --yes: deletes closed sessions from DB, prints count
    - sessions clear --state closed (no --yes): prompts for confirmation
    - sessions clear: no matching sessions -> prints "No sessions" message, does not prompt
    - sessions clear --state all --yes: deletes all sessions
    - db vacuum: server not running -> vacuums DB, prints success
    - db vacuum: server running -> prints error, exit 1
    - db vacuum: DB error -> prints error, exit 1
  </behavior>

2. Add `playwright-proxy-ctl = "playwright_mcp_proxy.ctl:main"` to `[project.scripts]` in `pyproject.toml`.

3. Create `playwright_mcp_proxy/ctl/__init__.py`:
   - Import `cli` from `.commands`
   - Define `main()` that calls `cli()`

4. Create `playwright_mcp_proxy/ctl/commands.py` with Click command groups. Use the patterns from 05-RESEARCH.md:
   - Top-level `@click.group() cli` with docstring "playwright-proxy-ctl: Manage the Playwright MCP Proxy server."
   - `@cli.command("health")`: GET `http://{settings.server_host}:{settings.server_port}/health`, print status and subprocess status. Catch `httpx.ConnectError` (print "Server is not running", exit 1) and `httpx.TimeoutException` (print "timed out", exit 1). Use `timeout=5.0`.
   - `@cli.group() sessions`: "Session management commands."
   - `@sessions.command("list")` with `--state` option (default None): GET `/sessions` with optional `state` query param. Print each session as `{session_id[:8]}... state={state} url={url}` then total count. Handle empty list. Catch connection errors.
   - `@sessions.command("clear")` with `--state` (default "closed", show_default=True) and `--yes` flag: Use `asyncio.run()` to call async function. Connect to DB via `aiosqlite.connect(str(settings.database_path))`. Count matching sessions. If 0, print "No sessions" and return. If `--yes` not set, use `click.confirm()` with `abort=True`. Execute DELETE. Commit. Print deleted count. Support `--state all` for deleting everything.
   - `@cli.group() db`: "Database maintenance commands."
   - `@db.command("vacuum")`: First check if server running via GET `/health` with `timeout=1.0` -- if responds, print error and exit 1. If `ConnectError`, proceed (server not running). Use `sqlite3.connect()` (stdlib, sync) to VACUUM. Print success with db path. Catch `sqlite3.OperationalError`.

   Import `settings` from `..config`. Import `httpx`, `click`, `asyncio`, `aiosqlite`, `sqlite3`.

   - Use `unittest.mock.patch` to mock `httpx.get` for HTTP-dependent commands (health, sessions list, db vacuum server check)
   - For `sessions clear` and `db vacuum`: create a temp SQLite DB, patch `settings.database_path` to point to it, seed with test data
   - For `sessions clear` tests: use `aiosqlite` to create sessions table and insert test rows, then verify deletion
   - For confirmation prompt testing: use `CliRunner.invoke(cli, [...], input="y\n")` for CliRunner input simulation
   - Minimum 9 test functions covering all behaviors

6. Run `uv pip install -e ".[dev]"` to install click dependency and register entry point.
